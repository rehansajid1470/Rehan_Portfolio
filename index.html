Featured Example: GUI Application Bug Fix

UI Freezing Bug in Tkinter Application

This common but critical bug in Tkinter applications causes the entire user interface to freeze, making the application completely unresponsive. The issue occurs when long-running operations or infinite loops run in the main thread.

Buggy Implementation 
# BUG: Inefficient update approach that freezes UI
def update_counter(self):
    while True:
        current_time = time.strftime("%H:%M:%S")
        self.time_label.config(text=current_time)
        time.sleep(1)  # This will freeze the GUI

The bug: Using an infinite loop with time.sleep() inside a GUI method blocks the main thread and freezes the application.

Fixed Implementation 
# FIXED: Efficient update method using after()
def update_time(self):
    current_time = time.strftime("%H:%M:%S")
    self.time_label.config(text=current_time)
    # Schedule the next update after 1000ms (1 second)
    self.root.after(1000, self.update_time)

The fix: Using Tkinter's after() method to schedule updates non-blockingly, keeping the UI responsive.
